HTML5 y su evolución

Es una evolución importante respecto a las versiones anteriores de HTML, introduciendo nuevas etiquetas y funcionalidades que permiten una mejor gestión del contenido multimedia, formularios avanzados, y mejoras en la semántica de la página web.


Etiquetas en HTML5

  1. <section>: Uso para agrupar contenido relacionado dentro de una página.
  2. <article>: Representa un bloque de contenido autónomo, como un artículo de un blog, una noticia o un post que puede existir independientemente del sitio web.
  3. <aside>: Define contenido relacionado con el contenido principal pero no esencial. Normalmente se utiliza para barras laterales, anuncios, o información complementaria.
  4. <header> y <footer>: Etiquetas para definir las cabeceras y pies de página.
  5. <nav>: Específica para las barras de navegación del sitio web.
  6. <figure> y <figcaption>: Agrupa elementos multimiedia y sus descripciones.

Buenas prácticas
  1. Utiliza elementos semánticos para organizar mejor la página. Esto hace que sea más comprensible para los navegadores, buscadores y lectores de pantalla.
  2. Evita el uso excesivo de divs ("divididos") y aprovecha las etiquetas como <section>, <article>, y <aside> para hacer que la página sea más semántica.
  3. Proporciona enlaces de navegación claros dentro de los elementos <header> y <footer> para mejorar la experiencia de usuario y la accesibilidad.
  4. No abuses de <aside>, puesto que este elemento tiene que contener información complementaria y no esencial para el contenido principal.
