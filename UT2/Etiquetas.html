Etiquetas básicas

Etiqueta <center>

Se utilizaba en verisones antiguas de HTML para centrar el contenido dentro de una página web, tanto texto como imagenes y otros elementos.

Está obsoleta en HTML5, y se debería evitar su uso. En lugar del center, hoy en día se utiliza CSS para centrar elementos.

Buenas prácticas
  • Siempre utiliza el atributo alt para garantizar una buena accesibilidad y SEO.
  • Evita insertar imagenes muy grandes sin reducir su medida, ya que eso puede afectar al rendimiento de la página.
  • Si las imagenes son responsive, utiliza porcentaje en vez de píxeles por width y height.


Etiqueta <div>
  • La etiqueta se utiliza para crear contenedores de bloques en una página HTML. Es una nueva etiqueta de nivel bloque,
    lo que significa que ocupa todo el ancho disponible y comienza una línea nueva.
  • No tiene significado semántico en si misma; simplemente agrupa otros elementos, como texto, imágenes u otras etiquetas y suele ser estilizada con CSS
  • Es útil para organizar y diseñar diferentes secciones de una página web.

Buenas prácticas
  • Uno de los errores más comunes es el uso excesivo de divs para cualquier cosa, cuando existen etiquetas semánticas
    más adequadas como <header>, <section>, <article>, etc..
  • Utiliza el div para agrupar elementos que forman una unidad lógica o visual dentro de la página.


Etiqueta <span>
  • Es una etiqueta inline (en línea) que sirve para estilizar o manipular pequeñas porciones de texto o contenido dentro de una página, este ocupa únicamente el espacio necesario.
  • También es una etiqueta no semántica, usada principalmente para aplicar estilos o funcionalidades a un grupo específico de elementos de un texto.

Buenas prácticas
  • Uso adecuado de <span> para estilizar texto en línea
  • Evitar el uso excesivo del elemento.
  • Combinarlo con CSS para mantener el código limpio.
  • Únicamente para elementos en línea.
  • Evitar abusar del elemento para crear contenedores sin necesidad.

Etiqueta <template>

La etiqueta se utiliza para definir fragmentos de HTML reutilizables que no se muestran inmediatamente cuando la
página se carga. El contenido dentro de uno no se añade al documento hasta que se manipula explícitamente con
JavaScript, cosa que permite crear plantillas que se pueden clonar e insertar al DOMO de manera dinámica.

Ejemplo:
Buenas prácticas
  • Mantén las plantillas simples y modulares: Organiza tu código dentro de la etiqueta de forma que puedas reutilizar fácilmente estas plantillas en diferentes secciones de la página.
  • Clona plantillas con JavaScript: La etiqueta no renderiza su contenido hasta que lo invocas con JavaScript, lo cual te da control total sobre cuando y como se tiene que añadir al DOMO.
  • Utiliza plantillas para generar contenido dinámicamente: Es muy útil en aplicaciones donde el contenido tiene que ser generado o actualizado en tiempo real, como listas, formularios o tarjetas de productos.
  • No incluyas contenido activo dentro del template: Los scripts u otros contenidos interactivos dentro del template no se ejecutarán hasta que el contenido sea añadido al DOMO, así que asegúrate de añadir los elementos interactivos una vez esté al documento.


Etiqueta <iframe>

Es una etiqueta HTML que permite insertar una página HTML dentro de otra página web. Es una manera más moderna y segura
de incrustar contenido externo, como otras páginas web, videos, formularios o mapas, dentro de una página principal.
Los iframes se utilizan a menudo para contenidos dinámicos como videos de YouTube, Google Maps, o integraciones de servicios externos.
La etiqueta se define dentro del cuerpo del documento y se puede personalizar con atributos como width, height, src, y frameborder.

Buenas prácticas
  • Evita el uso excesivo de iframes: El uso excesivo de iframes puede hacer que la página sea más lenta y más difícil de gestionar. Utilízalos solo cuando sea necesario (como para incrustar contenido externo).
  • Utiliza el atributo sandbox por seguridad: Cuando incrustas contenido otros lugares, utiliza el atributo sandbox para limitar las acciones que la iframe puede realizar dentro de tu sitio web, mejorando la seguridad.
  • Optimiza la altura y anchura: Ajusta la medida de la iframe para asegurarte que el contenido se vea correctamente en todas las medidas de pantalla, especialmente en dispositivos móviles.
  • Utiliza alternativas cuando sea posible: Cuando solo necesitas cargar contenido dinámico que puedes generar tú mismo (como un video o una imagen), considera cargarlo directamente en la página en lugar de utilizar un iframe.
  • Utiliza allowfullscreen para videos: Si estás incrustando videos, permite que se puedan visualizar en pantalla completa con el atributo allowfullscreen para mejorar la experiencia del usuario.
